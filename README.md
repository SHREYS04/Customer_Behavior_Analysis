# Customer_Behavior_Analysis
# Data Analytics Project

## Overview

This project demonstrates an end-to-end data analytics workflow using Python, PostgreSQL, and Power BI. The objective is to transform raw data into meaningful insights through data cleaning, exploratory analysis, SQL querying, interactive dashboards, and a final analytical report.

The project follows a complete analytics pipeline commonly used in real-world business environments.

---

## Dataset

The dataset contains structured data used for analysis and visualization.

The data is processed to:

* Understand its structure
* Handle missing values and duplicates
* Correct data inconsistencies
* Prepare it for analysis and reporting

## Project Workflow

### 1. Data Loading

* Imported the dataset into Python.
* Examined data types and overall structure.
* Verified dataset integrity.

### 2. Exploratory Data Analysis (EDA)

* Generated descriptive statistics.
* Identified trends and patterns.
* Created visualizations for better understanding.
* Detected outliers and anomalies.

### 3. Data Cleaning

* Removed duplicate records.
* Handled missing values.
* Corrected inconsistent data.
* Converted columns to appropriate data types.
* Prepared the dataset for further analysis.

### 4. SQL Analysis (PostgreSQL)

* Imported cleaned data into PostgreSQL.
* Wrote SQL queries to:

  * Filter records
  * Aggregate data
  * Perform joins
  * Generate business insights
  * Calculate KPIs

### 5. Power BI Dashboard

Built an interactive dashboard featuring:

* KPI cards
* Trend analysis
* Category-wise performance
* Charts and graphs
* Interactive filters and slicers

### 6. Final Report

Compiled the analysis into a report highlighting:

* Key findings
* Business insights
* Data-driven recommendations
* Dashboard summary
